Possible future software

Below are some proposed software ideas that I might create in the future.

Portable HTML Document standard

An alternative to PDF.
The difference from regular HTML is that it’s split into pages, and contains all the styling and images in the same file.
CSS styles can be updated while the file is open.
An example of a use for this is with e-books. A reader may want to change the font, text size or page colour.
Also, a user may right-click on an image to save it, as with a regular HTML file.
Other HTML features such as embedded audio and video can be used in the document.

New FuzeVD

Revive the old Fuze video downloader, switching to modern open-source libraries.
Would be available as a browser extension, Embedding a download button onto videos.

HTML Formatter

A simple tool to help in web development.

Bulky web frameworks may not be needed for smaller projects (such as fuze.page).
However, in HTML pages there are some repeating sections, most commonly a navigation bar.
With this tool, you could simply write an HTML comment, and it will add a code snippet from another file.

FuzeBox

This game is designed to be a sandbox where people can upload and share the maps, assets, and even gamemodes they create.

Single or multiplayer games can be hosted.

Theoretically, a diep.io clone or almost any 2D browser game imaginable could be re-created within this engine.

Think of it like a two-dimensional Garry's Mod.
Update: This project will be postponed until at least the full release of Warren Budget.
